# Auditoría SEO — ProductosVirales.com.ar

**Fecha:** 13 de abril de 2026  
**Dominio:** productosvirales.com.ar  
**Stack:** Next.js 16.2.3, React 19, Tailwind CSS v4  

---

## Resumen ejecutivo

**ProductosVirales tiene una base técnica sólida** — schema JSON-LD en productos y guías, sitemap dinámico bien priorizado, Server Components para todas las páginas principales, y buena jerarquía de headings. Sin embargo, **el sitio no está indexado en Google** (la búsqueda `site:productosvirales.com.ar` devuelve cero resultados), lo que invalida todo el trabajo SEO realizado hasta ahora.

**Las 3 prioridades que más impacto van a tener:**

1. **Resolver la indexación** — Sin esto, nada del resto importa. Falta `robots.ts`, no hay evidencia de que el sitemap esté registrado en Google Search Console, y posiblemente hay un issue de deploy/DNS.
2. **Agregar canonical URLs y viewport meta** — Faltan en la mayoría de páginas, lo que genera confusión para los crawlers.
3. **Crear contenido orientado a keywords de alto volumen** — El sitio tiene 6 tipos de página pero compite contra gigantes (MercadoLibre, TiendaNube, Nubimetrics) sin contenido diferenciado de blog/SEO.

**Evaluación general:** Fundaciones buenas, ejecución incompleta, indexación bloqueada.

## 2. Auditoría On-Page

### Problemas detectados

| Página | Problema | Severidad | Fix Recomendado |
|--------|----------|-----------|-----------------|
| Todas | No indexado en Google | **Crítica** | Verificar deploy, DNS, registrar sitemap en GSC, crear robots.ts |
| Home (`/`) | Sin metadata export propia | Alta | Agregar `generateMetadata` con title, description, canonical |
| Home (`/`) | Sin structured data | Alta | Agregar schema Organization + WebSite con SearchAction |
| Trending (`/trending`) | Sin canonical URL | Alta | Agregar `alternates.canonical` |
| Trending (`/trending`) | Sin OpenGraph ni Twitter card | Media | Agregar OG image y twitter:card |
| Categorías (`/categoria/[slug]`) | Sin `generateStaticParams()` | Alta | Agregar para pre-rendering de categorías conocidas |
| Categorías | Sin canonical URL | Alta | Agregar `alternates.canonical` dinámico |
| Categorías | Sin BreadcrumbList schema | Media | Agregar JSON-LD BreadcrumbList |
| Categorías | Sin OpenGraph images | Media | Agregar OG image por categoría |
| Productos (`/producto/[id]`) | Sin `generateStaticParams()` | Alta | Agregar al menos para productos featured |
| Productos | Sin canonical URL | Alta | Agregar `alternates.canonical` |
| Productos | Sin Twitter card | Media | Agregar `twitter: { card: "summary_large_image" }` |
| Productos | BreadcrumbList schema faltante | Media | Agregar JSON-LD con Home > Categoría > Producto |
| Guías (`/guias/[slug]`) | OG image no configurada | Media | Agregar imagen de guía en openGraph.images |
| Layout raíz | Sin viewport export | **Crítica** | Agregar `export const viewport = { width: 'device-width', initialScale: 1 }` |
| Layout raíz | Sin favicon/icons export | Media | Agregar icons en metadata |
| Todas las imágenes | `unoptimized={true}` | Alta | Remover para habilitar optimización de Next.js |

### Lo que está bien hecho

- H1 presente y relevante en todas las páginas principales
- Jerarquía de headings H1 > H2 > H3 correcta
- JSON-LD Product schema en páginas de producto (con Offer y precio)
- JSON-LD Article schema en guías (con author, publisher, dates)
- JSON-LD FAQPage cuando hay FAQ disponible
- Sitemap dinámico con prioridades inteligentes por visibilidad
- Internal linking sólido: header, footer, breadcrumbs, related products
- `rel="nofollow sponsored"` en links de afiliados (guías)
- Server Components en todas las páginas principales (buen SSR)
- Google site verification file presente

## Notas finales

El hallazgo más importante de esta auditoría es que **el sitio no está indexado**. Todo el trabajo de schema, metadata y contenido es inútil si Google no puede encontrar tus páginas. La prioridad número uno es resolver esto (robots.ts + GSC + verificar accesibilidad pública) y recién después avanzar con las optimizaciones on-page y la estrategia de contenido.

Una vez indexado, tu mayor ventaja competitiva es la **data en tiempo real de MercadoLibre** que ningún blog estático puede replicar. Usá eso para crear contenido dinámico que se actualice automáticamente y sea imposible de copiar.
